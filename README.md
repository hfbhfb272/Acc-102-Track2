# Acc-102-Track2
Questions and Audience: This project analyzes the long-term financial performance trend of Guizhou Moutai (stock code: 600519) from 2001 to the present, covering solvency, profitability, asset size and revenue growth. It provides reference materials for investment analysts, business managers and course evaluators regarding the company's operating conditions and investment value.
Data (source + access date + key fields):Data Source: WRDS platform CSMAR financial database (https://wrds-www.wharton.upenn.edu/)Data Acquisition Date: April 20, 2026Key Fields: Stock Code, Accounting Period, Reporting Type, Total Assets, Total Liabilities, Owner's Equity, Operating Revenue, Net Profit, etc.Filter Conditions: Stock Code is 600519, Annual Report, Accounting Period starts from January 1, 2001
Methods (main Python steps)：1. Connect to the WRDS remote database using the wrds library and extract historical financial data of Moutai using SQL statements.2. Use pandas to perform data cleaning and handle missing values, and calculate core financial indicators such as asset-liability ratio, ROA, ROE, and revenue growth rate.3. Use matplotlib to draw a 2×2 multi-subplot visualization to visually present the long-term trends of each financial indicator.4. Export the cleaned data as a CSV file and save the generated charts in the figures folder.
Key Findings：Guizhou Moutai has maintained a low and stable debt-to-asset ratio for a long time, with extremely low debt repayment risks. Its financial structure is sound. The ROA and ROE have remained at high levels for over two decades. The company has extremely strong and stable profitability. The total assets have achieved continuous and steady growth, and the business scale has been expanding. The revenue growth rate has experienced periodic fluctuations, but the long-term positive growth trend is clear, and the operational resilience is strong.
How to run：Open the Jupyter Notebook and run the code line by line in sequence. For File 1, it is a demonstration of data querying and extraction. For File 2, it is about using Python to obtain, clean, transform, analyze and present the data, and finally generate charts.The cleaned data is saved in the "data" folder, and the visual charts are saved in the "figures" folder.
